🛑 Importante: Não criar roteiro automático! 

O roteiro é criado apenas com base nos dados inseridos pelo usuário no modal “Criar Novo Roteiro”. Nada é gerado por padrão. 

 

✅ 1. Modal “Criar Novo Roteiro” 

Campo “Destino” é livre para digitação. 

Botão “Criar Roteiro” deve: 

Fechar o modal. 

Salvar os dados no banco PostgreSQL: 

postgres://viajey:viajey@viajey_viajey:5432/viajey?sslmode=disable 
  

Campos obrigatórios: 

nome_do_roteiro 

destino 

data_ida 

data_volta 

preferencias[] 

faixa_preco 

Redirecionar para a página /explorar. 

 

🌍 2. Página /explorar 

Pesquisa automática usando o destino salvo no banco. 

Consultar Google Places API para os tipos: 

lodging → Hotéis 

restaurant → Restaurantes 

tourist_attraction → Atrações 

Layout: 

Título: Explorando [DESTINO] 

Chips com as preferências do usuário 

Botões de filtro: Hotéis, Restaurantes, Atrações 

Cards gerados da API: 

Foto 

Nome 

Avaliação 

Endereço 

Preço 

Botões: 

“Ver detalhes” 

“+ Adicionar ao roteiro” → abre modal de alocação 

Mapa à direita com marcadores 

Ao clicar no marcador: exibir nome + botão “Ver detalhes” 

 

🔁 Navegação fluida entre /explorar e /roteiro/:id/planejamento 

Adicionar no topo de ambas as páginas botões de navegação: 

Se estiver em /explorar: botão “Ir para Planejamento” 

 → Redireciona para /roteiro/:id/planejamento 

Se estiver em /roteiro/:id/planejamento: botão “Voltar para Explorar” 

 → Redireciona de volta para /explorar 

Essa troca permite que o usuário explore locais e monte o roteiro alternando rapidamente entre as interfaces. 

 

📅 Modal: “Adicionar ao Roteiro” 

Aparece ao clicar em “+ Adicionar ao Roteiro” 

Exibe: 

Nome do local 

Dias da viagem (calculados com base na data_ida e data_volta do banco) 

Para cada dia: opções de período → manhã, tarde ou noite 

Ao confirmar: 

Salva o local no banco vinculado ao roteiro e ao dia/horário selecionado 

 

🗂️ Página /roteiro/:id/planejamento – Kanban 

Um quadro para cada dia da viagem (horizontal) 

Dentro de cada dia: três colunas fixas → manhã, tarde, noite 

Cards organizáveis por drag-and-drop 

Cada card representa um local adicionado 

Exibe: 

Foto 

Nome 

Tipo (ícone) 

Horário 

Botão de remover 

 

📍 Página /local/:place_id – Detalhes 

Ao clicar em “Ver detalhes” na página /explorar 

Exibir: 

Galeria de imagens (Google Photos) 

Nome, avaliação, endereço, descrição (se houver) 

Horário de funcionamento 

Caixa de clima (mock ok) 

Botão: “Adicionar ao roteiro” 

Mapa centralizado no local
